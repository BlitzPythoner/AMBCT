AMBCT (Automatic Manual Backup Creation Tool) is a command-line based backup utility that uses wimlib-imagex for creating reliable and highly compressed system or file backups.

It allows users to select drives or folders, define a target location, choose compression methods, and manage advanced options like integrity checks or automatic shutdown.

This tool combines automation and manual control – perfect for users who want efficiency without losing control over how their backups are created.

1. Execution

⬇ Download for Windows (x64) ⬇ Download for Windows (x32)

Double-click the downloaded .exe file. Admin rights will be requested automatically.

You'll arrive at the main menu of AMBCT.

2. Inputs

2.1 Drive selection

Select the drive or path you want to back up.

2.2 Target drive selection

Choose where the backup should be stored.

2.3 Compression method

Select one of four compression levels (None, XPRESS, LZX, LZMS).

2.4 Additional options

Add options like integrity check, solid mode, or shutdown after backup.

2.5 Disk speed measurement

The tool measures your target disk write speed to estimate backup time.

2.6 Overview & Confirmation

Check all selected options and confirm to start backup.

2.7 Backup process

Backup begins. You'll see a progress bar with time estimation.

2.7.1 Scanning

All files in the selected location are scanned and cataloged.

2.7.2 Backup Creation

Files are compressed and copied to the target location.

2.7.3 Verifying

The backup is verified to ensure all files were copied correctly.

3. Summary

AMBCT is a lightweight, command-line tool for creating reliable and space-efficient backups using wimlib-imagex. It guides the user through selecting source paths, choosing the target location, selecting a compression method, and enabling optional integrity checks.

Process at a glance: select source drive/folder → set target → choose compression and additional options → check write speed → review and confirm → start backup. During the process you will see scanning, creation, and verification stages.

Tip: verify the target path, ensure sufficient storage, and consider integrity checks before deleting old backups. Store created backups in a safe location.
